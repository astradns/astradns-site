# ADR-008: Estrategia Multi-Motor

- Estado: Aceptado
- Decisión: Definir la interfaz de engine desde el inicio y evolucionar motores por fases (Unbound, luego CoreDNS y PowerDNS).
- Motivo: Mantener el datapath estable y habilitar diversidad de backends.
- Consecuencia: Nuevos motores se agregan sin rediseñar proxy, métricas o logging.
