# ADR-004: Estratégia de Failover

- Status: Aceito
- Decisão: Usar failover nativo do engine no MVP e adicionar circuit breaker no agent no pós-MVP.
- Motivo: Entregar confiabilidade incremental sem atrasar a entrega inicial.
- Consequência: O plano de evolução preserva observabilidade e melhora a resiliência por fases.
