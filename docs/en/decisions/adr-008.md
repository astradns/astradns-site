# ADR-008: Multi-Engine Strategy

- Status: Accepted
- Decision: Define an engine interface from day one and roll out engines in phases (Unbound first, then CoreDNS and PowerDNS).
- Why: It keeps the datapath stable while enabling backend diversity.
- Consequence: New engines can be added without redesigning proxy, metrics, or logging layers.
